# 서버

<h2 id="server">
  서버 정보
</h2>

### 구조

| FIELD | TYPE | DESCRIPTION |
|-------|------|-------------|
| id | string | 서버의 ID |
| name | string | 서버의 이름 |
| flags | integer | 서버의 플래그 |
| intro | string | 서버의 짧은 설명 |
| desc | string | 서버의 긴 설명 |
| votes | integer | 서버의 하트 수 |
| category | [Category](#category)[] | 서버의 카테고리 |
| invite | string | 서버의 초대링크 |
| state | [State](#state) | 한국 디스코드봇 리스트에서의 서버의 상태 |
| vanity | ?string | 봇의 VANITY URL |
| bg | ?string | 봇의 배경 이미지 주소 |
| banner | ?string | 봇의 배너 이미지 주소 |
| icon | ?string | 서버의 아바타 해시 |
| members | integer | 서버의 멤버 수 |
| emojis | [Emoji](#emoji)[] | 서버의 이모지 |
| boostTier | integer | 서버의 부스트 티어 |
| owner | [User](./유저#user)[] | 서버의 관리자 |

<h2 id="category">
 서버 카테고리 (Category)
</h2>

- 커뮤니티
- IT & 과학
- 봇
- 친목
- 음악
- 교육
- 연애
- 게임
- 오버워치
- 리그 오브 레전드
- 배틀그라운드
- 마인크래프트

<h2 id="state">
  서버의 상태 (State)
</h2>

- `ok` 정상
- `reported` 일시 정지
- `blocked` 강제 삭제
- `archived` 잠금 처리 (지원 종료)

<h2 id="flags">
  서버 플래그(flags)
</h2>

| VALUE | DESCRIPTION |
|-------|-------------|
| 0 << 0 | 없음 |
| 1 << 0 | 공식 봇 |
| 1 << 2 | 한국 디스코드봇 리스트 인증된 봇 |
| 1 << 3 | 파트너 |
| 1 << 4 | 디스코드 인증된 봇 |
| 1 << 5 | 프리미엄 |
| 1 << 6 | 제1회 한국 디스코드봇 리스트 해커톤 우승자 봇 |

<h2 id="emoji">
  서버 이모지(Emoji)
</h2>

| FIELD | TYPE | DESCRIPTION |
|-------|------|-------------|
| id | string | 이모지의 ID |
| name | string | 이모지의 이름 |
| url | string | 이모지의 원본 URL |

#### Request Header

| FIELD | TYPE |DESCRIPTION |

|-------|------|-------------|
| userID | string | 확인할 유저 아이디 |
